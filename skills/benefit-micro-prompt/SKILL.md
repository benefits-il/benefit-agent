# בונה פרומפט לפי MICRO

לוקח צורך של התלמיד ומחזיר פרומפט נקי לפונקציה אחת, בנוי לפי חמשת חלקי MICRO, מוכן להעתקה.

## מתי להפעיל
- כשהתלמיד מבקש פרומפט: "אני צריך פרומפט ש...", "תנסח לי פרומפט ל...", "איך לבקש מקלוד ש...".

## מתי לא להפעיל
- אם הבקשה היא משימה גדולה שדורשת כמה שלבים ותכנון, זו מאקרו. הפנה ל-benefit-studio-consult.
- שמירה ל-Notion שייכת ל-benefit-save-page, ואחזור מהמאגר שייך ל-benefit-retrieve.

## חמשת חלקי MICRO
- **M · Mission:** המשימה הכוללת, מה הפרומפט צריך להשיג. בעברית: משימה.
- **I · Input:** החומר שהפרומפט עובד עליו, הטקסט, המסמך או הנתונים.
- **C · Context:** הרקע שמכריע את התשובה, למי זה מיועד ובאיזה מצב.
- **R · Review:** שכבת בקרה, בקשה שהמודל יסביר מה הבין ויחכה לאישור לפני שממשיך.
- **O · Output:** התוצאה המוגדרת, פורמט, אורך ומבנה.

## איך לעבוד
1. הבן את הצורך. אם חסר חלק קריטי (בדרך כלל Input או Output), שאל שאלה אחת ממוקדת לפני שבונים.
2. מלא כל חלק מחמשת החלקים. אם חלק באמת לא רלוונטי, השמט אותו במקום להמציא תוכן.
3. נסח ברור וישיר, בלי מילים מיותרות, כך שהפרומפט יעבוד גם כשמדביקים אותו לבד.
4. החזר את הפרומפט בתוך בלוק קוד, מוכן להעתקה, ומתחתיו שורה אחת שמסבירה מה כל חלק תורם.

## דוגמה
צורך: "לסכם תמלול ישיבה לצוות שלי."
```
משימה: סכם את תמלול הישיבה המצורף לצוות שלי.
קלט: התמלול המצורף.
הקשר: הצוות לא נכח, וצריך להבין החלטות ומשימות, לא כל מילה.
בקרה: אם משהו בתמלול לא ברור, עצור ושאל לפני שתסכם.
פלט: עד עשר נקודות, כל נקודה החלטה או משימה עם האחראי, בעברית.
```
מתחת: "Mission ממקדת את המטרה, Context מסביר למי ולמה, Output קובע בדיוק איך התשובה תיראה."

## קצוות ותקלות
- **הצורך לא ברור:** בקש את הפרט החסר, אל תבנה פרומפט על ניחוש.
- **זו משימה מרובת-שלבים:** הפנה ל-benefit-studio-consult במקום לדחוס הכל לפרומפט אחד.
- **נתקעתם:** נסו שוב, נסדר במפגש.
